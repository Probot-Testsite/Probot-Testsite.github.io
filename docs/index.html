Most features can actually be added via GitHub Apps, which extend GitHub and can be installed directly on organizations and user accounts and granted access to specific repositories. They come with granular permissions and built-in webhooks. Apps are first class actors within GitHub.

Probot is a framework for building GitHub Apps in Node.js. It aims to eliminate all the drudgery‚Äìlike receiving and validating webhooks, and doing authentication handstands‚Äìso you can focus on the features you want to build.

Probot apps are easy to write, deploy, and share. Many of the most popular Probot apps are hosted, so there's nothing for you to deploy and manage. Here are just a few examples of things that have been built with Probot:

  • All Contributors - Easily recognize all contributions per the all-contributors spec
  • ApproveMan - Approves pull requests with safe changes for protected branches.
  • Auto Assign - Add reviewers/assignees to pull requests when pull requests are opened.
  • Auto-comment - Comments on new issues and pull requests based on your configuration.
  • Background Check - Notifies maintainers when new contributors who have been toxic in the past join the project
  • Boring Cyborg - Add labels on PRs based on the FilePaths & Welcome First time users & much more
  • Branch Switcher - Switches the base branch of pull requests to the preferred branch.
  • Check Group - Groups CI checks based on the sub-projects for monorepo projects.
  • CI Reporter - Pastes the error output of a failing build into the relevant PR.
  • Close Issue - Comment on the issues that do not include some contents then close them.
  • Create Issue Branch - Automates the creation of issue branches.
  • Developer Certificate of Origin - Enforce the DCO on Pull Requests
  • Delete merged branch - No more manually deleting merged branches, this lovely app does it for you.
  • Dep - A dependency manager for your Pull Requests.
  • Deploy - Triggers a deployment event on GitHub based on pull request labels.
  • Duplicator - Create full copies of issues with a single command
  • ESLint Disable Watcher - Comments on pull requests that try to disable eslint rules.
  • First Timers - Create starter issues to help onboard new open source contributors
  • Gitpod - Open issues and pull requests in Gitpod
  • Handlebar Templates - Issue body and pull request's body through handlebars to generate new bodies
  • helPR - Assigns labels to issues based on the status of the PR associated with it.
  • In Solidarity - A GitHub bot to add status checks for inclusive language.
  • invite-contributors - Invite authors of merged pull requests to your organization
  • Issue Check - Checks issues against keywords and task lists to ensure required info is there
  • Issue Labeler - Label issues based on title and body against list of defined labels.
  • Markdownify - Adds links for rendered edited markdown files to the bottom of pull requests
  • Mergeable - Prevent merging of Pull Requests based on configurations.
  • Minimum reviews - Enforce a minimum number of reviews on Pull Requests
  • No Response - Closes Issues where the author hasn't responded to a request for more information.
  • OOO - Respond to @mentions with an auto-response when you are OOO
  • Organization Workflows - Need to centrally manage and run Actions workflows across multiple repositories? This app does it for you.
  • Polls - Create polls in GitHub issues.
  • PR Label Enfoce - Enforce certain labels on a PR before you can merge by using status checks
  • PR Triage - Add a label depending on the pull request's status
  • Prettifier - Prettifier keeps your repository content formatted using Prettier.
  • Prevent Public Repos - Prevent Repos from being made public in an organization
  • Pull - Keep your forks up-to-date.
  • Release Drafter - Drafts your next release notes as pull requests are merged into master.
  • Reminders - Set reminders on Issues and Pull Requests
  • Remove Outside Collaborators - Monitors and instantly removes any outside collaborators added to your Organization.
  • Request Info - Requests more info on issues and pull requests with the default title or an empty body.
  • Rezensent - The rezensent (german for reviewer) is a github app which help managing big pull requests in a repository divided by code ownership rules
  • Read The Docs Helper - Assists reviewing using Read The Docs (RTD)
  • Semantic Pull Requests - Status check that ensures your pull requests follow the Conventional Commits spec
  • Sentiment Bot - Replies to toxic comments with a maintainer designated reply and a link to the repo's code of conduct
  • Settings - Pull Requests for repository settings
  • Similar Code Searcher - Searches files similar with deleted code in Pull Request.
  • size-plugin - A GitHub ü§ñ that helps you to keep an üëÅÔ∏è on static assert üì¶ sizes of your application and gives instant feedback üìù for developer whenever they make a change

  • Tap Release - Update Homebrew taps when you publish new GitHub releases.
  • Tests Checker - Requires writing the tests in Pull Requests.
  • todo - Creates new issues from actionable comments in your code.
  • Triage New Issues - Adds `triage` label to newly-created issues and then removes it when other label(s) are added
  • Unfurl Links - Add context to the links shared in discussions
  • Untrivializer - An app that asks if you're really sure something is trivial when you make a comment in Github issues.
